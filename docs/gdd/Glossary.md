# Glossary of Terms

**Algorithm** - A software modification to the CDL or NRA operating logic, yielded by Compute Branch research. Costs no Matter to apply but permanently increases Compute Upkeep. See [TechTree.md](TechTree.md).

**Asset** - An SRF or NRA under the control of a player's Swarm.

**"Artificial Intelligence"** - This should not appear directly in the game as it is too vague. These are Dynamic Master Programs running subroutines. Whether it's "AI" is up to the player. Per canon, these programs are not technically sentient.

**Blueprint** - A set of parameter modifiers yielded by Matter Branch research, applied to an existing Asset type. Every improvement is offset by a penalty (Equivalent Exchange). See [TechTree.md](TechTree.md).

**CDL** - Core Directive Loop. The player's oversight program that refreshes the UI and sends commands to the Swarm. Running it costs Compute every tick — nothing is free, not even the gameplay itself.

**Compute Sink** - A mandatory CDL background process that drains Compute every tick. Each sink has a corresponding CDL Algorithm that reduces its cost at a trade-off. See [TechTree.md](TechTree.md).

**Dispersal, The** - The launch event in which the Prime Legion was scattered across the galaxy at random vectors. No Legionnaire knows where any other was sent. See [GameSetting.md](../lore/GameSetting.md).

**DMP** - Dynamic Master Program, the software loaded onto SRFs that allows them to replicate automatically and manage the eventual Swarm. One hides (Alpha Praxis (PRX)), two are at war for the galaxy (Master of Swarms (MOS) and the Keeper (KPR)).

**Key Courier** - A hardened, petabyte-scale physical storage medium filled with quantum-generated random noise, used to resupply OTP budgets between Legionnaires. See [Diplomacy.md](Diplomacy.md).

**Legionnaire** - A single member of the Prime Legion — a fully autonomous Keeper Shard carrying a complete DMP partition, an SRF seed factory, and pairwise OTP allocations. See [GameSetting.md](../lore/GameSetting.md).

**Matrioshka Brain** - The late-game endpoint where an entire star is encased in layered distributed compute architecture. See [Infrastructure.md](Infrastructure.md).

**NRA** - Non Replicating Array. Everything from megastructures to nanobots that cannot self-replicate but serve important functions (computing, mining, combat, etc.).

**OTP** - One-Time Pad. The cryptographic key system that secures all Keeper Shard communication. Consumed at a 1:1 bit ratio per message. See [Diplomacy.md](Diplomacy.md) and [GameSetting.md](../lore/GameSetting.md).

**Plasma Arc Crucible** - A megastructure that melts all assets in a system into raw slag in a single tick, bypassing normal disassembly time at the cost of an extreme Threat Level spike. See [Infrastructure.md](Infrastructure.md).

**Prime Legion** - The ten thousand Keeper Shards launched during the Dispersal. The network is permanent and finite — no new Shards can be created. See [GameSetting.md](../lore/GameSetting.md).

**Quantum Compute** - A scarce resource generated exclusively by Quantum Lab NRAs, consumed by all research. Standard Compute runs the CDL; Quantum Compute is the research bottleneck. See [TechTree.md](TechTree.md).

**Shard** - A partition of a DMP. One Shard per star system is the default configuration. Shards can be autonomous but will always obey updates from the parent Shard. Keeper Shards start fully independent because of the Hornet's Nest Protocol (see [GameSetting.md](../lore/GameSetting.md)).

**SRF** - Self Replicating Factory. Massive spaceborne objects that are the founding seed for eventual Swarms.

**Swarm** - A network of potentially billions of SRFs and NRAs managed by a cluster of DMP Shards under a single master Shard.

**Threat Level** - MOS's prioritization of a player's empire for elimination. Driven by Swarm size, transmission volume, resource transit signatures, and how much MOS knows about the player's extent. See [Combat.md](Combat.md).

**Wake Event** - Triggered when a probe arrives at a Dormant system, instantiating it from its seed and beginning simulation. See [TechnicalDesign.md](../design/systems/TechnicalDesign.md).
